# Nhost Todo Application

A production-ready full-stack Todo application built with **Nhost** (PostgreSQL, GraphQL, Authentication) and **React**.

![Todo App Screenshot](https://via.placeholder.com/800x450?text=Todo+App+Screenshot)

## Overview

This application demonstrates a complete integration with Nhost's Backend-as-a-Service platform, featuring:

- **Authentication**: Email/Password sign-up and sign-in
- **Database**: PostgreSQL with auto-generated GraphQL API
- **Real-time Operations**: CRUD operations via GraphQL mutations
- **Modern UI**: Clean, dark-themed interface with animations

## Tech Stack

| Layer | Technology |
|-------|------------|
| Frontend | React 19, React Router, Framer Motion |
| Styling | Tailwind CSS, Shadcn/UI Components |
| Backend | Nhost (PostgreSQL + Hasura GraphQL) |
| Authentication | Nhost Auth (Email/Password) |
| API | GraphQL (auto-generated by Hasura) |

## Setup Instructions

### Prerequisites

- Node.js 18+ (or 20+)
- npm or yarn
- Nhost account (free tier available)

### 1. Create Nhost Project

1. Go to [nhost.io](https://nhost.io) and sign up for a free account
2. Create a new project:
   - Click "New Project"
   - Choose a name (e.g., "todo-app")
   - Select a region closest to you (e.g., `eu-central-1`)
   - Wait for the project to be provisioned (~2-3 minutes)

3. Note your project credentials:
   - **Subdomain**: Found in project settings (e.g., `abcd1234`)
   - **Region**: The region you selected (e.g., `eu-central-1`)

### 2. Create Database Schema

1. In Nhost Dashboard, go to **Database** → **SQL Editor**
2. Run the following SQL to create the `todos` table:

```sql
-- Create todos table
CREATE TABLE public.todos (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    title TEXT NOT NULL,
    is_completed BOOLEAN DEFAULT false,
    created_at TIMESTAMPTZ DEFAULT now(),
    user_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
);

-- Create index for user queries
CREATE INDEX idx_todos_user_id ON public.todos(user_id);
CREATE INDEX idx_todos_created_at ON public.todos(created_at DESC);

-- Enable Row Level Security
ALTER TABLE public.todos ENABLE ROW LEVEL SECURITY;

-- Policy: Users can only see their own todos
CREATE POLICY "Users can view own todos" ON public.todos
    FOR SELECT USING (auth.uid() = user_id);

-- Policy: Users can insert their own todos
CREATE POLICY "Users can insert own todos" ON public.todos
    FOR INSERT WITH CHECK (auth.uid() = user_id);

-- Policy: Users can update their own todos
CREATE POLICY "Users can update own todos" ON public.todos
    FOR UPDATE USING (auth.uid() = user_id);

-- Policy: Users can delete their own todos
CREATE POLICY "Users can delete own todos" ON public.todos
    FOR DELETE USING (auth.uid() = user_id);
```

### 3. Configure Hasura Permissions

1. Go to **Data** → **todos** table → **Permissions** tab
2. Add permissions for the `user` role:

**Select Permission:**
- Row select permissions: `{"user_id":{"_eq":"X-Hasura-User-Id"}}`
- Column select permissions: `id, title, is_completed, created_at`

**Insert Permission:**
- Row insert permissions: `{"user_id":{"_eq":"X-Hasura-User-Id"}}`
- Column insert permissions: `title`
- Column presets: `user_id` → `X-Hasura-User-Id`

**Update Permission:**
- Row update permissions: `{"user_id":{"_eq":"X-Hasura-User-Id"}}`
- Column update permissions: `is_completed`

**Delete Permission:**
- Row delete permissions: `{"user_id":{"_eq":"X-Hasura-User-Id"}}`

### 4. Configure Environment Variables

Update `/app/frontend/.env` with your Nhost credentials:

```env
# Nhost Configuration
REACT_APP_NHOST_SUBDOMAIN=your-actual-subdomain
REACT_APP_NHOST_REGION=eu-central-1
```

### 5. Install Dependencies & Run

```bash
cd frontend
yarn install
yarn start
```

The app will be available at `http://localhost:3000`

## Database Schema

### `todos` Table

| Column | Type | Default | Description |
|--------|------|---------|-------------|
| `id` | UUID | `gen_random_uuid()` | Primary key, auto-generated |
| `title` | TEXT | - | Todo title (required) |
| `is_completed` | BOOLEAN | `false` | Completion status |
| `created_at` | TIMESTAMPTZ | `now()` | Creation timestamp |
| `user_id` | UUID | - | Foreign key to auth.users |

## GraphQL Queries & Mutations

### Fetch All Todos

```graphql
query GetTodos {
  todos(order_by: { created_at: desc }) {
    id
    title
    is_completed
    created_at
  }
}
```

### Create Todo

```graphql
mutation CreateTodo($title: String!) {
  insert_todos_one(object: { title: $title }) {
    id
    title
    is_completed
    created_at
  }
}
```

### Update Todo (Toggle Completion)

```graphql
mutation UpdateTodo($id: uuid!, $is_completed: Boolean!) {
  update_todos_by_pk(
    pk_columns: { id: $id }
    _set: { is_completed: $is_completed }
  ) {
    id
    title
    is_completed
    created_at
  }
}
```

### Delete Todo

```graphql
mutation DeleteTodo($id: uuid!) {
  delete_todos_by_pk(id: $id) {
    id
  }
}
```

## Troubleshooting

### "Invalid or expired token" error
- Ensure your Nhost credentials are correct in `.env`
- Try signing out and signing back in

### GraphQL errors
- Check that the `todos` table exists with correct schema
- Verify Hasura permissions are set up correctly

### Empty todo list after sign-in
- Make sure Row Level Security policies are configured
- Check browser console for GraphQL errors
